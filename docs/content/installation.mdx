# Установка и обновление

## Требования

- Клиент **exteraGram** или **AyuGram** для Android версии **12.1.1** и выше.
- **Собеседнику плагин не нужен** — стрик считается локально, по вашей истории
  переписки. Исключение одно: чтобы принять приглашение в общий
  [серийчик](/features/streak-pet#как-завести-серийчика), плагин должен стоять
  и у него.

## Установка

<Steps>

### Скачайте файл плагина

Файл `tg-streaks.plugin` лежит в закреплённом сообщении официального
[Telegram-канала плагина](https://t.me/n08i40k_extera/3). Там же — описание и
ссылка на исходный код. Этот пост обновляется при каждом релизе, поэтому всегда
берите файл именно из него, а не ищите отдельные посты про конкретные версии.

### Откройте скачанный файл

Нажмите на скачанный `.plugin` файл — exteraGram или AyuGram сами покажут окно
установки. Заходить в настройки плагинов отдельно не нужно.

<Screenshot
  file="installation/install-dialog-light.jpg"
  dark="installation/install-dialog-dark.jpg"
  caption="Диалоговое окно установки плагина после открытия .plugin файла"
/>

### Подтвердите установку

После подтверждения клиент подключит плагин. При первом запуске он ещё
докачает два файла — **движок** (`classes.dex`) и **ресурсы** (`resources.zip`
с анимациями и картинками уровней). На это время внизу ненадолго появится
уведомление о скачивании. Это нормально и происходит один раз.

</Steps>

## Как понять, что всё получилось

- В списке плагинов клиента появилась карточка **Streaks** с
  [настройками](/settings).
- В [меню личного чата](/chat-menu) появились пункты плагина.
- Стрик покажется не сразу — только после **3 дней** взаимной переписки. Если
  его пока нет, это нормально: [как стрик появляется](/features/streaks#как-стрик-появляется).

## Проверка целостности файлов

<Callout type="info">
  Оба докачиваемых файла проверяются по контрольной сумме SHA-256. Если сумма
  не совпала — например, скачивание оборвалось или файл был подменён — плагин
  сам себя отключит и покажет диалог с объяснением. Лечится это просто:
  переустановите плагин на стабильном интернете, и файлы скачаются заново. См.
  также [FAQ про этот диалог](/faq).
</Callout>

<Callout type="info">
  Если файл вообще не удалось скачать (например, нет интернета), появится
  отдельный диалог **«Не удалось скачать данные плагина»** с кнопками
  **«Повторить»** и **«Отмена»** — это не проблема с целостностью, а просто
  неудачная попытка загрузки.
</Callout>

## Обновление

Плагин сам проверяет GitHub Releases на новую версию при старте клиента (можно
выключить в [настройках](/settings#проверка-обновлений)). Как устроена эта
проверка и что означает диалог «Перезапустить клиент?» — на отдельной странице
[«Обновления плагина»](/updates#автопроверка-новой-версии).
